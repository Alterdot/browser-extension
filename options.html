Extension Settings
  Here you can configure your connections to the Alterdot and IPFS networks and more. Setting up your browser extension correctly is a must if you want to take advantage of a fully decentralized experience. If you just want to browse the Alterdot web without hosting a local wallet or an IPFS node you can skip straight to the section about Alterdot Domain Identifiers.
Wallet RPC Port Set the port used by your desktop wallet for JSON-RPC calls. It is found under rpcport in your alterdot.conf file, by default the desktop wallet sets it to 31050.
Wallet RPC Username Set the username used by your desktop wallet for JSON-RPC calls. It is found under rpcuser in your alterdot.conf file, by default the wallet generates a random username.
Wallet RPC Password Set the password used by your desktop wallet for JSON-RPC calls. It is found under rpcpassword in your alterdot.conf file, by default the wallet generates a random password. This is not your wallet passphrase!
IPFS Local Gateway Port The port used by your local IPFS gateway. This can be found in the IPFS Companion under gateway, by default it gets set to 8080, if that one is used then 8081. If you're using Brave this port is usually set to 48084.
Alterdot Domain Identifiers
  Whenever you try to access an Alterdot domain our extension looks for one of several special identifiers that let it know that you are indeed looking for an Alterdot domain. There are three default identifiers: .adot and .a which are added at the end of the domain name as well as a. which is added in front. For example, by searching for either bitcoin.adot, bitcoin.a or a.bitcoin you will be taken to the bitcoin domain registered on our BlockchainDNS. The last format interferes with normal domains named a, e.g. a.com, so if you want to access them, disable Intercepted Search in the main extension popup.
  Important! If Intercepted Search is disabled you have to add / at the end of the domain request like bitcoin.adot/ or bitcoin.a/. Beware that a.bitcoin/ format is not supported.
Custom Access Identifier Create your own custom identifier here, by default ".chain" can also be used for addressing, change it to anything you want.
Use Debugging Enabling this results in numerous debug messages being logged in the console. If you encounter any problem while using the browser extension, you should report it to the developers. Using this Debug feature might be requested in order to correctly identify the problem.
